What it does

Everything a server-grade agent does.
Running in your pocket.

⚙

Full Agent Runtime

Cross-compiled Rust core. Agent loop, tool execution, memory, channels — everything from the upstream project, running as a native Android subprocess.

🔒

Private by Default

No telemetry. No cloud dependency. Your data stays in your app sandbox. Bring your own LLM API keys.

🚀

Foreground Service

Stays alive across screen-off, Doze mode, and app kills. Starts on boot. Survives Android's aggressive memory management.

🌐

Multi-Channel

Telegram, Discord, webhooks, direct API — your agent responds where you are. Configure channels from the web dashboard at localhost:18789.

📄

Sandboxed Execution

Shell commands, file operations, and HTTP fetches all run inside the app's SELinux sandbox. Whitelist only what you trust.

💬

Any LLM Provider

OpenAI, Anthropic, Ollama, OpenRouter, local models — 28+ providers supported. Switch in config, zero code changes.

SQLite Vector Memory

Persistent memory with vector search. The agent remembers context across sessions. Zero external databases.

memory.db → sqlite + vector embeddings
receipts/ → tool call audit logs
config.toml → editable in-app
Under the hood

Four layers. One binary.

The Rust runtime does the work. Android keeps it alive.

Layer 1

Rust Runtime

Simon|AI binary compiled for aarch64-linux-android via NDK. Agent loop, gateway, memory, tools, channels.

features = ["agent-runtime",
"gateway", "memory-sqlite",
"tool-shell", "tool-http"]
Layer 2

Foreground Service

Kotlin service spawns the binary as a subprocess. START_STICKY keeps it alive. Crash = auto-restart.

ProcessBuilder(binary, "daemon",
"-p", "18789").start()
Layer 3

Dashboard UI

Jetpack Compose app talks to the local gateway REST API. Full native Android experience, no WebView wrappers.

http://127.0.0.1:18789
→ ApiClient → Compose UI
